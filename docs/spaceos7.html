产品介绍

什么是 SpaceOS 7？

SpaceOS 7 是纯自研的混合内核架构操作系统。2026 年，我们掌握了 UEFI 技术后，决定完全不依赖外部代码，自己从头写操作系统。现在完成了一半，还在继续开发。

项目起源

2026 年，我们完全掌握了 UEFI 技术，启动了这个自研项目。没有外部代码，没有依赖，UEFI 引导、内核加载都是我们自己写的。

设计理念

"完全自研、混合内核、极致性能"。混合内核架构结合微内核和宏内核的优点，既保证稳定性，又提供高性能。

核心特性

SPFS 2.0-Next 文件系统

自研的高性能硬盘文件系统 SPFS 2.0-Next，为硬盘存储优化，提供卓越的文件操作性能和可靠性。

  • 硬盘高性能优化
  • 日志功能支持
  • 文件权限管理
  • 低延迟读写

SPUFS 1.0 文件系统

自研的 USB 存储设备文件系统 SPUFS 1.0，专为 USB 存储设备设计，支持热插拔和即插即用。

  • USB 存储设备优化
  • 热插拔支持
  • 即插即用
  • 兼容 USB 2.0/3.0

内核重构

中间引导和内核完全重构，全部手搓自研。SpaceOS 7 的内核从零开始构建，不依赖任何外部代码，真正实现了完全自研。

  • 完全自研的引导程序，并没有使用 Limine 和 grub 这样的开源引导程序
  • 自研的内核架构，抛弃 Uinxed-Kernel 内核转为纯自研
  • 伙伴系统内存管理，Slab 分配器
  • 高效的进程调度，支持进程间 IPC，支持用户态

Ring 3 桌面环境

桌面环境运行在 Ring 3，提升系统安全性。通过将桌面环境运行在用户空间，即使桌面环境崩溃也不会影响内核。

  • 桌面环境用户空间运行，稳定性更高
  • 内核空间与用户空间隔离
  • 更高的系统安全性（因为有ring3）
  • 更好的稳定性（因为桌面环境一崩内核不会炸）

SMP 多核支持

实现 SMP (Symmetric Multi-Processing) 多核支持。SpaceOS 7 完美支持多核处理器，能够充分利用多核性能。

  • 多核调度支持
  • 多核同步机制
  • 多核内存管理
  • 多核中断处理

xHCI 控制器

SpaceOS 7 支持 xHCI (eXtensible Host Controller Interface)。SpaceOS 7 提供了完整的 xHCI 驱动，支持 USB 3.0 及以上版本。

  • USB 3.0 支持
  • USB 2.0 向后兼容
  • 多设备支持
  • 高带宽传输

空间 API

支持空间 API，提供更丰富的系统调用接口。空间 API 是我们自研的系统调用接口，提供了更强大的功能。

  • 丰富的系统调用
  • 高性能 IPC
  • 灵活的通信机制
  • 安全的权限控制

性能提升

相比 SpaceOS 6，性能大幅提升，系统更加流畅。通过优化内核和各种组件，SpaceOS 7 在各个方面都有显著的性能提升。

  • 更快的系统启动
  • 更快的应用启动
  • 更快的文件操作
  • 更流畅的用户体验

技术细节

混合内核架构

SpaceOS 7 采用创新的混合内核架构，结合微内核和宏内核的优点。

  • 微内核组件用户空间化
  • 关键服务内核空间运行
  • 高效的 IPC 机制
  • 完善的权限管理

纯手写 UEFI 库

完全自研的引导程序，从零开始构建 UEFI 库，不依赖任何外部代码。

  • 纯手写 UEFI 库
  • 高半地址内核加载
  • UEFI 引导支持
  • 深入理解 UEFI 规范

内存管理

先进的内存管理机制，包括伙伴系统和 Slab 分配器。

  • 伙伴系统（Buddy System）
  • Slab 分配器
  • 4KB 和 2MB 大页支持
  • 完整的虚拟内存
  • 硬件辅助内存保护

进程调度

先进的进程调度算法，确保公平性和效率。

  • CFS 完全公平调度器
  • 实时调度支持
  • 多核负载均衡
  • 节能调度

设备驱动

丰富的设备驱动支持，覆盖各种硬件设备。

  • 存储驱动（IDE/AHCI/NVMe/USB/SCSI）
  • 显示驱动
  • 网络驱动
  • 音频驱动
  • 输入驱动
  • xHCI USB 驱动

空间 API

自研的系统调用接口，提供强大的功能和高效的 IPC。

  • 丰富的系统调用
  • 高性能 IPC
  • 灵活的通信机制
  • 安全的权限控制
