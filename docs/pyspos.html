项目介绍

什么是 PySpOS？

PySpOS 是一个完全使用 Python 编写的操作系统模拟环境，由 GoutouStdio 工作室研发。它不是真正的操作系统内核，而是在 Python 运行时环境中模拟操作系统行为的实验性项目。

PySpOS 旨在通过 Python 的高级抽象，探索操作系统设计的核心概念，包括进程管理、文件系统、系统调用等。项目从 2022 年的"MP4 架构狗头 OS"起步，经过多次迭代，现已发展为功能完善的 v3.0 版本。

架构设计

PySpOS 采用模块化架构设计，核心组件包括：内核（kernel.py）、文件系统（fs.py）、OTA 更新模块（ota.py）、SPF 解析器（parse_spf.py）等。每个模块职责明确，便于维护和扩展。

系统支持 A/B 双槽位升级机制，确保更新失败时可回滚。OTA 更新系统支持从云端拉取更新包，自动验证完整性并安装。

命令行界面

PySpOS 提供类似 Linux 的命令行界面，支持目录导航、文件操作、应用运行等功能。用户可以通过简单的命令与系统交互，体验操作系统的基本操作。

核心特性

OTA 在线更新

支持从云端检查更新、下载更新包、验证完整性、A/B 槽位升级。OTA 系统支持多种下载方式（requests、urllib、curl），确保在各种环境下都能成功获取更新。

SPF 可执行文件

SPF（Space Portable Format）是 PySpOS 的自研可执行文件格式。SPF 解析器支持简单的系统调用，如 putchar() 输出和 exit() 退出，适合教学与实验。

文件系统模拟

提供完整的文件系统 API，支持目录创建、文件读写、复制移动等操作。基于宿主机的文件系统，实现类 Unix 的目录导航和管理功能。

ROOT 权限管理

支持 ROOT 权限切换，通过配置文件控制权限状态。提供 testroot 命令验证权限，确保系统操作的安全性。

模块化架构

清晰的模块化设计，包括 kernel.py、main.py、fs.py、printk.py、logk.py 等核心模块。每个模块职责单一，便于开发、测试和维护。

A/B 槽位升级

支持 A/B 双槽位设计，更新安装到非活动槽位，确保升级失败可回滚。槽位状态记录在 current_slot 文件中，启动时自动切换。

命令行界面

提供丰富的命令行工具，包括 ls、cd、echo、osver、python 等。支持运行 apps 目录下的 Python 应用程序，扩展系统功能。

完全开源

遵循 MIT 许可证，完全开源免费。代码托管在 GitHub，欢迎社区贡献和反馈。适合操作系统初学者学习与实践。

技术细节

内核模块

kernel.py 是 PySpOS 的主内核，负责处理用户输入、执行命令、管理应用运行。内核启动时显示 ASCII 艺术 Logo，初始化各子系统。

kernel.loop() - 主循环，处理用户输入

SPF 解析器

parse_spf.py 实现 SPF 文件格式的解析和执行。支持 putchar() 和 exit() 系统调用，适合编写简单的演示程序。

putchar("Hello"); exit(0);

OTA 更新

ota.py 处理 OTA 更新逻辑，包括版本检查、下载、验证、安装。支持从 https://pyspos.us.ci/ota/ 获取更新。

compare_versions(v1, v2) - 版本比较

文件系统

fs.py 提供文件系统操作 API，包括 list_dir、create_file、read_file、delete_file 等。基于 Python 的 os 和 shutil 模块实现。

fs.create_file("test.txt", "content")

配置管理

btcfg.py 管理启动配置，包括 ROOT 权限、启动器锁定状态等。配置文件存储在 etc/bootcfg.json。

btcfg.load_bootcfg() - 加载配置

应用系统

apps 目录包含系统应用，如 calc.py（计算器）、hello.py（示例程序）等。通过 open 命令运行应用。

open hello - 运行 hello.py
