项目介绍

什么是 PySpOS？

PySpOS 是用 Python 写的操作系统模拟器。不是真的操作系统，是在 Python 环境里模拟操作系统行为。

从 2022 年的"MP4 架构狗头 OS"开始，现在迭代到 v3.0 版本。能运行命令、管理文件、支持 OTA 更新。

架构设计

模块化设计：内核（kernel.py）、文件系统（fs.py）、OTA 模块（ota.py）、SPF 解析器（parse_spf.py）。每个模块独立，好维护。

A/B 双槽位升级：更新装到非活动槽位，验证后再切换。失败了可以回滚到之前的版本。

命令行界面

类 busybox 命令行，支持 cd、ls、echo 等命令。可以运行 apps 目录下的 Python 程序。

核心特性

OTA 在线更新

从云端检查更新、下载、验证、安装。支持 A/B 槽位升级，更新失败可回滚。支持 requests、urllib、curl 多种下载方式。

SPF 可执行文件

SPF（Space Portable Format）是自研可执行文件格式。目前支持 putchar() 输出和 exit() 退出，适合教学实验。

文件系统模拟

完整的文件系统 API，支持目录创建、文件读写、复制移动。基于宿主机文件系统，实现类 Unix 目录导航。

ROOT 权限管理

支持 ROOT 权限切换，通过配置文件控制权限。提供 testroot 命令验证权限。

模块化架构

kernel.py（内核）、main.py（入口）、fs.py（文件）、printk.py 和 logk.py（日志）。每个模块职责单一。

A/B 槽位升级

双槽位设计，更新装到非活动槽位，验证成功后切换。槽位状态记录在 current_slot 文件中。

命令行界面

提供 ls、cd、echo、osver、python 等命令。支持运行 apps 目录下的 Python 程序。

完全开源

MIT 许可证，开源免费。代码在 GitHub，欢迎提 issue 和 PR。适合操作系统初学者学习。

技术细节

内核模块

kernel.py 是 PySpOS 的主内核，负责处理用户输入、执行命令、管理应用运行。内核启动时显示 ASCII 艺术 Logo，初始化各子系统。

kernel.loop() - 主循环，处理用户输入

SPF 解析器

parse_spf.py 实现 SPF 文件格式的解析和执行。支持 putchar() 和 exit() 系统调用，适合编写简单的演示程序。

putchar("Hello"); exit(0);

OTA 更新

ota.py 处理 OTA 更新逻辑，包括版本检查、下载、验证、安装。支持从 https://pyspos.us.ci/ota/ 获取更新。

compare_versions(v1, v2) - 版本比较

文件系统

fs.py 提供文件系统操作 API，包括 list_dir、create_file、read_file、delete_file 等。基于 Python 的 os 和 shutil 模块实现。

fs.create_file("test.txt", "content")

配置管理

btcfg.py 管理启动配置，包括 ROOT 权限、启动器锁定状态等。配置文件存储在 etc/bootcfg.json。

btcfg.load_bootcfg() - 加载配置

应用系统

apps 目录包含系统应用，如 calc.py（计算器）、hello.py（示例程序）等。通过 open 命令运行应用。

open hello - 运行 hello.py

兼容性要求

Windows

Windows 7 及以上版本，推荐使用 Windows 以获得最佳兼容性

Linux

主流发行版，如 Ubuntu、Debian、Fedora 等

macOS

macOS 10.14 及以上版本

Python

Python 3.7 或更高版本，需要安装 requests 库
